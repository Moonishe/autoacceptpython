# Auto-Accept Dota 2 (Python)

Скрипт для автоматического принятия матча (кнопка **Accept**) в Dota 2.

Мониторит цвет пикселя в фиксированной точке экрана, где появляется кнопка Accept, и при совпадении с эталонным цветом — кликает по ней через `SendInput`.

## Особенности

- **0 внешних зависимостей** — только стандартная библиотека Python + `ctypes` (Win32 API)
- **Без инжектов и модификации игры** — не лезет в память `dota2.exe`, не меняет файлы игры
- **Клик только когда Dota 2 в фокусе** — не сработает в другом приложении
- **Debounce 8 секунд** — защита от повторных кликов
- **Глобальный хоткей стопа** — `Ctrl+Shift+Q`
- **Логирование** — все действия пишутся в `auto_accept.log`

## Требования

- Windows
- Python 3.7+
- Dota 2 (любой режим окна, разрешение 1920×1080 по умолчанию)

## Установка

```bash
git clone https://github.com/Moonishe/autoacceptpython.git
cd autoacceptpython
```

Дополнительно ничего ставить не нужно — скрипт использует только стандартную библиотеку.

## Использование

### 1. Калибровка (один раз)

Наведи курсор мыши на центр кнопки Accept в Dota 2 и не двигай его 5 секунд:

```bash
python auto_accept.py calibrate
```

Скрипт запишет координаты и цвет пикселя в `auto_accept_config.json`.

> **Важно:** калибруй когда кнопка Accept **реально видна** на экране. Если записать цвет фона — будут ложные срабатывания.

### 2. Запуск (перед поиском матча)

```bash
python auto_accept.py
```

Скрипт начнёт мониторить точку экрана. Как только появится кнопка Accept — кликнет автоматически.

### 3. Остановка

- `Ctrl+Shift+Q` — глобальный хоткей
- Или просто закрыть консоль

### Просмотр конфига

```bash
python auto_accept.py status
```

## Конфиг

После калибровки создаётся `auto_accept_config.json`:

```json
{
  "x": 960,
  "y": 540,
  "color": [10, 80, 40],
  "tolerance": 35,
  "interval_ms": 350,
  "debounce_ms": 8000,
  "require_dota_focus": true
}
```

| Параметр | Описание | По умолчанию |
|----------|----------|-------------|
| `x`, `y` | Экранные координаты центра кнопки | — |
| `color` | Эталонный RGB цвет кнопки `[r, g, b]` | — |
| `tolerance` | Допуск отклонения цвета (сумма по 3 каналам = `tolerance × 3`) | 35 |
| `interval_ms` | Как часто проверять пиксель (мс) | 350 |
| `debounce_ms` | Пауза после клика (мс) | 8000 |
| `require_dota_focus` | Кликать только если окно Dota 2 активно | true |

Если калибровка не срабатывает — уменьши `tolerance` или перекалибруй при реальной кнопке.

## Безопасность (VAC)

Скрипт **не вызывает VAC-бан**, потому что:

- Не инжектит в память `dota2.exe`
- Не модифицирует файлы игры
- Использует только `SendInput` (симуляция ввода) и чтение пикселя экрана

Формально это нарушение Terms of Service, но Valve исторически не банивает за макросы без игрового преимущества.
